N皇后

​ 按照国际象棋的规则，皇后可以攻击与之处在同一行或同一列或同一斜线上的棋子。n 皇后问题 研究的是如何将 n 个皇后放置在 n×n 的棋盘上，并且使皇后彼此之间不能相互攻击。即给定一个正整数n，表面要在nxn的棋盘上放n个皇后，并且这些皇后不能在同行、同列和同对角线上，求出一共有多少种放置的方案。并且N皇后问题的时间复杂度位O(n!)。

数组记录位置

​ 我们可以用一个数组path记录每个皇后的放置位置，path[i]表示第 i 行的皇后放在了哪一列。然后遍历第一行的皇后可能放置的所有列，向下递归展开，找到所有可能的方案。

int path[10]{}; //path[i]表示第i行的皇后放在了哪一列

bool check(int n, int row, int col)
{
    //检查当前列有没有被其他行占用或者在一条对角线上
    for(int i = 0; i < row; i++)
        if(path[i] == col || abs(i - row) == abs(col - path[i]))
            return false;

    return true;
}

int dfs(int n, int curRow)
{
    if(curRow == n)
    {
        return 1; //[0, n - 1]行都找到了一个合适的位置，找到了一种解决方案
    }
    int ans = 0;
    //找目前所有可放置的列进行展开
    for(int j = 0; j < n; j++)
    {
        //找到一个没有被占用的列，检查该列能不能放
        if(check(n, curRow, j))
        {
            path[curRow] = j;
            ans += dfs(n, curRow + 1);
        }
    }
    return ans;
}

位运算优化

​ 使用数组的方式记录每个皇后的位置信息，每次放置一个新皇后的时候都需要遍历一遍这个数组来判断该位置是否能放，使用位信息来存储每个皇后的位置信息就能对这一步骤进行优化。

​ 我们可以使用一个变量，用他的二进制位来表示每一列的状态，从低位到高位中第 i 位表示第 i 列是否可以放置皇后（规定0表示可以放置，1表示不可以放置），而且当前行的操作产生的影响可以通过这个变量传递到下一行去，因此不需要开一个数组表示。

int dfs(int limit, int cur, int left, int right)
{
    //所有位置都不能再放皇后了，说明此时已经找到一个合法的放置方案了
    if(cur == limit)
        return 1;
    
    int path = cur | left | right; //找到当前行所有不能放置的地方(不能放为1，能放为0)
    int choice = limit & (~path); //翻转一下，让可放置的位置为1，不可放置的位置为0
    
    int ans = 0;
    //找到所有可选择的位置
    while(choice)
    {
        int place = choice & -choice; //找到最右侧的1，即找到一个可放置的地方
        choice -= place; //choice减去最右侧的1，让下个循环接着找到下一个1

        //当前位置放好，记录状态调用子过程（因为棋盘的位置和二进制的高低位是反着的，所以对右下角的影响要向左移，对左下角的影响要向右移）
        ans += dfs(limit, cur | place, (left | place) >> 1, (right | place) << 1);
    }
    
    return ans;
}

int totalNQueens(int n) {
    int limit = (1 << n) - 1; //让limit的低n位变成1，作为限制条件
    
    return dfs(limit, 0, 0, 0);
}

求二叉树的高度

​ 二叉树中的高度是左右子树的最大高度中的最大值，并且从树的高度从底部向上是增加的。

int TreeHeight(TreeNode* root)
{
    if(root == nullptr) return 0;
    
    int lHeight = TreeHeight(root->left);
    int rHeight = TreeHeight(root->right);
    
    return max(lHeight, rHeight) + 1; //树的高度 = max(左子树高度，右子树高度) + 1
}

//自底向上搜索，从叶子节点开始向上返回信息
TreeNode* lowestCommonAncestor(TreeNode* root, TreeNode* p, TreeNode* q) {
    if(root == nullptr || root == p || root == q) return root; //边界条件
    
    TreeNode* l = lowestCommonAncestor(root->left, p, q); //查看p或者q是否在左子树上
    TreeNode* r = lowestCommonAncestor(root->right, p, q); //查看p或者q是否在右子树上
    
    if(l && r) return root; //如果p和q分布在该节点的左右子树上，那么该节点就是最近公共祖先
    return l ? l : r; //否则说明p和q在该节点的同一侧，返回找到的结果
}

判断完全二叉树

​ 下面代码的思路就是从左到右层序遍历二叉树，如果空节点后面有非空节点，则一定不是完全二叉树；如果遇到空节点后面就没有节点了，那么是完全二叉树。

bool check(Node* root)
{
    bool meetNull = false;
    queue<Node*>q;
    q.push(root);
    while(!q.empty())
    {
        Node* node = q.front();
        q.pop();
        
        if(node == nullptr)
            meetNull = true; //遇到空节点就将该变量设为true
        else
        {
            if(meetNull) return false; //如果已经遇到了空节点，后面有遇到了非空节点，则不是完全二叉树
            q.push(node->l);q.push(node->r);
        }
    }
	//未发现异常就说明是完全二叉树
    return true;
}

判断平衡二叉树

​ 根据平衡二叉树的性质可知，平衡二叉树的每一棵子树的左右子树的高度差不能大于1。因此我们就可以利用前面的求树的高度的代码，判断每棵子树的左右高度之差是否大于1。如果发现某棵子树的左右子树高度之差大于1，则不是平衡二叉树；如果遍历了整棵树也没有发现不合法的情况，则是平衡二叉树。

bool isBalanceTree = true; //全局遍历

int TreeHeight(TreeNode* root)
{
    if(root == nullptr) return 0;
    
    int lHeight = TreeHeight(root->left);
    int rHeight = TreeHeight(root->right);
    
    //发现某棵子树的左右子树高度差大于1，就标记为false
    if(abs(lHeight - rHeight) > 1)
        isBalanceTree = false;
    
    return max(lHeight, rHeight) + 1; //树的高度 = max(左子树高度，右子树高度) + 1
}

位运算

​ 在计算机中，一切数据都是以二进制存储的，所以计算机很擅长处理二进制的数据，而位运算就是直接对整数在内存中的二进制位进行操作，因此其执行效率非常高。本章节会对常用到位运算进行一些总结，下面涉及到的代码均为C++代码。

lowbit操作

​ lowbit操作可以快速求出数字x的二进制的最低位的1所表示的数，下面举个例子：

  • 001010011000 —> 对应lowbit(x) == 1000
  • 001010011010 —> 对应lowbit(x) == 10

​ 这个操作的实现也非常简单：

int lowbit(int x)
{
    return x & -x;
}

​ 在计算机中，通常使用补码来表示有符号整数，对于数字x来说，他的负数-x的补码是将x的二进制位取反后再加一得到的，这样以来x和-x的二进制形式只有从最低位的1到低位的0是相等的，其余高位都是相反的，因此二者进行与操作就会得到最低位的1。

​ 有了lowbit这样一个快速找到最低位1的操作后，我们也可以用来快速求数字x的二进制表示中有多少个1：

int lowbit(int x)
{
    return x & -x;
}

int getOneNum(int x)
{
    int cnt = 0;
    //x每次减去最右侧的1，直到变成0位置，使用lowbit可以快速求出1的个数
    for(int i = x; i != 0; i -= lowbit(i))
        cnt++;
    return cnt;
}

整数二分

二分法用于在有序序列中快速查找，由于每次都会跳过一半的数据，所以时间复杂度为O(logn)。下面写一个通用的整数二分模板，该模板会返回序列中第一个>=目标数target的数据的位置。

所以这是一个万能模板，会返回序列中第一个>=目标target的数的索引值，查找不到就返回数组长度。不论是找唯一数据的索引值、重复数据的左边界、大于target的第一个数的索引值（target不是数组的的某个值，但在数组的最大最小值之间），都可以直接用这个模板。

需求 写法 如果不存在
≥ x 的第一个元素的下标 lowerBound(nums, x) 结果为 n
> x 的第一个元素的下标 lowerBound(nums, x + 1) 结果为 n
< x 的最后一个元素的下标 lowerBound(nums, x) − 1 结果为 −1
≤ x 的最后一个元素的下标 lowerBound(nums, x + 1) − 1 结果为 −1
需求(数组下标从0开始) 写法
< x 的元素个数 lowerBound(nums, x)
≤ x 的元素个数 lowerBound(nums, x+1)
≥ x 的元素个数 n−lowerBound(nums, x)
> x 的元素个数 n−lowerBound(nums, x+1) 
//闭区间写法[l, r]
int lowerBound(vector<int>nums, int target)
{
    //这里我们要返回的索引值范围是[left, right]，都是闭区间，所以当left > right时该区间为空，说明查找结束
    int left = 0, right = nums.size() - 1;
    while(left <= right)
    {
        //防止left + right数据溢出
        int mid = left + (right - left) / 2;
        if(nums[mid] >= target)right = mid - 1;
        else left = mid + 1;
    }
    return left;
}



//开区间写法(l, r)
int lower_bound(vector<int>& nums, int target) 
{
    int left = -1, right = nums.size(); // 开区间 (left, right)
    while (left + 1 < right) { // 区间不为空
        //防止下标溢出
        int mid = left + (right - left) / 2;
        if (nums[mid] >= target) {
            right = mid; // 范围缩小到 (left, mid)
        } else {
            left = mid; // 范围缩小到 (mid, right)
        }
    }
    // 循环结束后 left+1 = right
    // 此时 nums[left] < target 而 nums[right] >= target
    // 所以 right 就是第一个 >= target 的元素下标
    return right;
}

质数筛

给定一个数字n，返回1~n范围内的所有质数的数量。

这个问题如果遍历从2到n，再分别判断每个数是不是质数，那时间复杂度是O(n$\sqrt{n}$)，是很容易超时的，而质数筛就比较快了。

共有埃氏筛和线性筛(欧拉筛)两种方法，前者时间复杂度为O(nlog(logn))，后者顾名思义时间复杂度为O(n)。一般情况下埃氏筛够用了，但是欧拉晒能表现的更好。

//埃氏筛的做法可以简单理解为把每一个质数的倍数都排除掉，最后留下来的就是质数
int getPrimeCount(int n)
{
    int prime[N];//记录所有的质数
	bool visit[N];//false表示为质数，true表示为合数，初始时里面的值均为false
    
    int cnt=0;
    for(int i = 2; i <= n; i++)
    {
        //第一次进入循环的i为2
        if(!visit[i])
        {
            prime[cnt++] = i;//找到了一个质数，存到数组里
            //遍历4，6，8，10，12...，将2的整数倍都排除
            for(int j = i * i; j <= n; j += i)
                visit[j] = true;
        }
    }
    return cnt;
}

高精度加

问题：两个长度为1000（或者更长，以至于无法用某一个类型表示它）的大正整数A和B相加，输出A+B的结果。

显然这么多位的数字，我们无法用任何一个数值类型变量存储，所以就需要使用高精度加方法来模拟计算。高精度加就是使用一个数组存储大整数的每一位，然后模拟加法运算。

#include <vector>
#include <string>
#include <iostream>
#include <algorithm>
using namespace std;

int main()
{
    string a, b;
    cin>>a>>b;
    
    //输入，因为要从个位开始加，所以逆序存储大整数
    vector<int>va, vb;
    for(int i = a.size() - 1; i >= 0; i--) va.push_back(a[i] - '0');
    for(int i = b.size() - 1; i >= 0; i--) vb.push_back(b[i] - '0');
    
    //计算，向前进位
    vector<int> ans;
    int t = 0; //t承担进位，最低位没有进位，初始值为0
    for(int i = 0 ; i < va.size() || i < vb.size(); i++)
    {
        if(i < va.size()) t += va[i];
        if(i < vb.size()) t += vb[i];
        ans.push_back(t % 10);
        t /= 10; //如果当前位相加的值 >= 10则t/10 == 1，表示向前进位，否则t/10 == 0，不会向前进位
    }
    //判断最高位是否还需要向前进位
    if(t) ans.push_back(t);
    
    //逆序输出
    for(int i = ans.size() - 1; i >= 0; i--)
        cout << ans[i];
    cout << endl;
    
    return 0;
}

滑动窗口

滑动窗口使用前提：

  1. 连续子数组/子串。（注意不是子序列，必须是连续的子串或者子数组）
  2. 有单调性，即所求的目标会随着窗口的变化单调变化。例如正数数组，或者负数数组，窗口内数组的和会随着数组的大小变化单调变化；但是如果数组内有正有负的话，就不满足单调性了，就无法使用滑动窗口解决。

定长窗口

定长窗口的题目求解目标一般明确指出要求长度为k的符合要求的连续子数组或者子串。可以直接从0到n - 1枚举所有右端点，枚举所有以i位置为右端点的窗口，从中找到符合要求的答案。

下面是算法模板：

int solution(vector<int>& nums, int k)
{
    int n = nums.size(), left = 0, ans = 0;
    //枚举所有右端点
    for(int i = 0; i < n; i++)
    {
        //右端点进入窗口
        ...;//计算
        
        //因为要求长度为k，所有窗口大小为k时才符合要求
        if(i - left + 1 < k)
            continue;
        
        //此时窗口大小已经为k了，就找到了一个长度为k的窗口
        ...;//计算该窗口，更新结果ans
        
        left++;//左端点向前移动
    }
    return ans;
}

动长窗口

动长窗口的题目一般不会对子数组或者子串的长度进行限制，一般会要求出满足某种限制条件的子数组或子序列，这种情况下窗口的大小就是不固定的。

下面是算法模板：

int solution(vector<int>& nums, int k)

    //枚举所有右端点
    for(int i = 0; i < n; i++)
    {
        //右端点进入窗口
        ...;//计算

        {
            //左端点移出窗口，直到该窗口内地子数组或者子序列满足限制条件

            left++;

        //此时的窗口是满足题目中限制条件的
        ...;//计算该窗口，更新结果ans
    }
    return ans;
}

矩阵对角线

​ 使用该模板可以快速遍历矩阵的对角线，假设我们的矩阵大小为m x n，即m行n列。

​ 首先来看第一种情况，对角线是左上到右下方向的。对于每一条对角线上的元素mat[ i ] [ j ]，i - j都是相同的值，于是我们可以发现这样一个式子：

k = i - j + n，这个 k 表示从右上到左下方向上第 k 条对角线（下图中红色数字就是k的值），k的取值是[1, m + n - 1]，所以我们只需要遍历k的所有取值就能遍历所有的对角线。

​ 对于第k条对角线来说，我们只需要确定元素的列坐标 j ，也就能根据等式 k = i - j + n 求出行坐标 i ，根据等式可以判断，i最小时j最小，i最大时j最大：

  • i 最小值为0时， minj = n - k。但是要保证minj >= 0，所以 minj = max(n - k, 0)
  • i 最大值为m - 1时， maxj = m - 1 - k + n。但要保证 maxj <= n - 1，所以maxj = min(m - 1 - k + n, n - 1)

​ 这样以来我们就可以在遍历每条对角线的时候求出 j 的取值范围，进而根据 k = i - j + n 求出 i ，从而找到该对角线上的所有元素mat[ i ] [ j ]。下面看看代码怎么写：

void matDiagonal(vector<vector<int>>& mat)
{
    //确定行数和列数
    int m = mat.size(), n = mat[0].size();
    
    //遍历所有对角线，k的取值范围为[1, m + n - 1]
    for(int k = 1; k < m + n; k++)
    {
        //根据 j = i - k + n 确定该对角线上 j 的取值范围
        int minj = max(n - k, 0); //i = 0时j最小，但时不能小于0
        int maxj = min(m - 1 - k + n, n - 1); //i = m - 1时j最大，但是不能大于n - 1
        
        //遍历对角线上的所有元素
        for(int j = minj; j <= maxj; j++)
        {
            int x = mat[j + k - n][j]; //根据公式 i = j + k - n 求出对应的行坐标 i
        }
    }
}

​ 下面介绍第二种情况，对角线是右上到左下方向的。对于每一条对角线上的元素mat[ i ] [ j ]，i + j都是相同的值，于是我们可以发现这样一个式子：

k = i + j，这个 k 表示从左上到右下方向上第 k 条对角线（下图中红色数字就是k的值），和上面一种情况不同的是，此时k的取值是[0, m + n - 2]，所以我们只需要遍历k的所有取值就能遍历所有的对角线。

​ 对于第k条对角线来说，我们只需要确定元素的列坐标 j ，也就能根据等式 k = i + j 求出行坐标 i ，根据等式可以判断，i最小时j最大，i最大时j最小：

  • i 最大值为m - 1时， minj = k - m + 1。但要保证 minj >= 0，所以minj = max(k - m + 1, 0)

  • i 最小值为0时， maxj = k。但是要保证minj <= n - 1，所以 maxj = min(k, n - 1)

​ 这样以来我们就可以在遍历每条对角线的时候求出 j 的取值范围，进而根据 k = i + j 求出 i ，从而找到该对角线上的所有元素mat[ i ] [ j ]。下面看看代码怎么写：

void matDiagonal(vector<vector<int>>& mat)
{
    //确定行数和列数
    int m = mat.size(), n = mat[0].size();
    
    //遍历所有对角线，k的取值范围为[0, m + n - 2]
    for(int k = 0; k < m + n - 1; k++)
    {
        //根据 j = k - i 确定该对角线上 j 的取值范围
        int minj = max(k - m + 1, 0); //i = m - 1时j最小，但时不能小于0
        int maxj = min(k, n - 1); //i = 0时j最大，但是不能大于n - 1
        
        //遍历对角线上的所有元素
        for(int j = minj; j <= maxj; j++)
        {
            int x = mat[k - j][j]; //根据公式 i = k - j 求出对应的行坐标 i
        }
    }
}

前缀和

​ 前缀和可以在**O(1)**的时间复杂求出子数组nums[left, right]的和，常见的有一维前缀和跟二维前缀和，下面分别看一下两种情况的算法模板：

一维前缀和

​ 设长度为n的数组nums（数组下标从0开始），存储前缀和的数组为preSum。规定：

  • preSum[0] = 0

  • preSum[1] = preSum[0] + nums[0] == nums[0]

  • preSum[2] = preSum[1] + nums[1] == nums[0] + nums[1]

  • preSum[3] = preSum[2] + nums[2] == nums[0] + nums[1] + nums[2]

  • …

  • preSum[n] = preSum[n - 1] + nums[n - 1] == nums[0] + nums[1] + nums[2] + ... + nums[n - 1]

​ 即preSum[i + 1]是nums数组中[0, i]范围内的元素之和。于是我们可以推导出nums的子数组nums[left, right]的元素之和为：

preSum[right + 1] - preSum[left]。

​ 假设我们要求子数组为nums[2, 4]的和，对应值为preSum[5] - preSum[2]。

​ 其中preSum[5] = nums[0] + nums[1] + nums[2] + nums[3] + nums[4]。

​ preSum[2] = nums[0] + nums[1] 。

​ 可以清晰的看到，将二者相减便可得到nums[2] + nums[3] + nums[4]范围的子数组之和。

int subarraySum(vector<int>& nums, int left, int right) 
{
    int n = nums.size();
    //nums数组长度为n，前缀和数组长度就要为n + 1，且preSum[0] = 0
    vector<int> preSum(n + 1, 0);
    
    //根据公式preSum[i + 1] = preSum[i] + nums[i]计算前缀和
    for(int i = 0; i < n; i++)
        preSum[i + 1] = preSum[i] + nums[i];
	
    //直接根据公式求出子数组的和
    return preSum[right + 1] - preSum[left];
}

​ 其实前缀和数组也可以定义为如下形式：

  • preSum[0] = nums[0]

  • preSum[1] = preSum[0] + nums[1] == nums[0] + nums[1]

  • preSum[2] = preSum[1] + nums[2] == nums[0] + nums[1] + nums[2]

  • preSum[3] = preSum[2] + nums[3] == nums[0] + nums[1] + nums[2] + nums[3]

​ 即preSum[i]是nums数组中[0, i]范围内的元素之和，此时子数组nums[left, right]的元素之和为：preSum[right] - preSum[left - 1]。但是由于出现了left - 1，但是left是可能为0的，所以这种写法需要特殊处理一下left == 0的情况。所以还是推荐使用上面的写法，不需要特殊判断left == 0的情况。

二维前缀和

​ 二维的整体思路和一维是类似的，但是情况稍微会复杂一点，但是也可以找到对应的公式运算。二维前缀和要求的是一个二维矩阵中由左上角顶点(r1, c1)和右下角顶点(r2, c2)所组成的矩形内的元素之和，就是下图中蓝色区域内的元素之和。

​ 于是我们定义sum[r + 1] [c + 1]为左上角为(0, 0)，右下角为(r, c)的矩形区域的元素和，这样定义的原因和一维前缀和一样。如果使用sum[r] [c]表示该区域的元素和时，当左上角为(0, 0)时需要特殊处理，否则会出现负数坐标，我们这样定义就省去了特殊判断(0, 0)的麻烦。

求任意矩形区域的前缀和

​ 下面看一下怎么求左上角为(r1,c1)，右下角为(r2,c2)的任意矩形的区域元素和。绿色和蓝色区域分别是左上角顶点和右下角顶点所表示的前缀和，我们要求的就是中间的红色部分：

​ 单独把我们要求的部分提取出来，可以发现要求的部分可以看作蓝色部分减去下图中两个黑色的区域，这两个区域分别可以用坐标为(r2,c1-1)、(r1-1,c2)的前缀和表示出来，但是要注意左上角红色部分被减了两次，所以要加回来，不能多减。

​ 所以(r1,c1)和(r2,c2)所表示的矩形区域的前缀和公式为：

​ target = sum[r2+1] [c2+1] - sum[r2+1] [c1] - sum[r1] [c2+1] + sum[r1] [c1]

二维前缀和数组的初始化

​ 我们要求的是坐标(r,c)的二维前缀和，可以利用前面已经求出过的前缀和来求解。如下图所示，我们要求的是红色区域，但是蓝色区域和绿色区域是已知的。所以我们可以把蓝色区域和绿色区域先相加，再减去重复的部分，最后再加上(r,c)本身的值，就能得到红色区域的元素和。

​ 公式为（不要忘记sum[r+1] [c+1]表示的才是(r,c)的前缀和）：

​ sum[r+1] [c+1] = sum[r+1] [c] + sum[r] [c+1] - sum[r] [c] + mat[r] [c]

代码

​ 下面看一下代码实现：

int sumRegion(vector<vector<int>>& matrix, int row1, int col1, int row2, int col2) {
    int m = matrix.size(), n = matrix[0].size(); //获得矩阵大小
    vector sum(m + 1, vector<int>(n + 1)); //构建二维前缀和数组，注意原数组是m x n的，前缀和数组要是(m+1)x(n+1)的
    
    sum[0][0] = 0; //(0,0)位置初始化为0，和一维中令sum[0]=0原理一样
    for(int i = 0; i < m; i++)
        for(int j = 0; j < n; j++)
            sum[i + 1][j + 1] = sum[i + 1][j] + sum[i][j + 1] - sum[i][j] + matrix[i][j]; //用公式初始化
    //按公式直接返回
    return sum[row2 + 1][col2 + 1] - sum[row2 + 1][col1] - sum[row1][col2 + 1] + sum[row1][col1];
}

差分

​ 前缀和可以用O(1)的时间复杂度求出任意子区域的和；而差分可以实现用O(1)的时间复杂度将任意子区域内的元素都加上一个常数c，实现对象数组整体的修改。差分和前缀和是存在某种联系的，即数组 a 的差分的前缀和就是数组 a，这么说肯看起来有点困惑，下面会详细讲一下。和前缀和类似，差分一分为一维差分和二维差分，前者是修改子数组，后者则是修改二维矩阵的子区域。

一维差分

​ 首先我们定义长度为n的原数组a（下标从0开始），及其差分数组d，前面已经说过差分数组的前缀和是原数组a，所以规定差分数组d满足下面的式子：

  • d[0] = a[0]
  • d[1] = a[1] - a[0]
  • d[2] = a[2] - a[1]
  • d[3] = a[3] - a[2]
  • …
  • d[n - 1] = a[n - 1] - a[n - 2]

​ 下面我们对差分数组d求一下前缀和看看会怎么样：

  • d[0] = a[0]
  • d[0] + d[1] = a[1]
  • d[0] + d[1] + d[2] = a[2]
  • …
  • d[0] + d[1] + … + d[n - 1] = a[n - 1]

​ 可以神奇的发现对差分数组求前缀和就能得到对应原数组的内容。根据式子发现，如果我们对差分数组中的d[i]进行修改，那么这个改动会影响a[i] ~ a[n]，因为a[i] ~ a[n]的值都依赖d[i]求出来的。如果d[i] = d[i] + c，那么a[i] ~ a[n]也都会加c，这就是差分数组实现修改范围的值的关键所在了。

​ 假设我们想实现将原数组a的[left, right]范围内的数都加上常数c的修改，经过多轮操作后返回最终的数组。如果每次操作都遍历一遍的话，那么每次时间复杂度就是O(n)，我们使用差分的话，就能把遍历操作优化为对差分数组修改两个值，这样就能将操作的时间复杂度将为O(1)。

​ 不卖关子，我们要做的操作就是：

  • d[left] += c
  • d[right + 1] -= c

​ 具体原理如下图所示，前面已经说过，对d[left]加c，会使得原数组中a[left] ~ a[n - 1]都加c；但是我们只想要a[left] ~ a[right]加c，于是就再让d[right + 1]减去c，消除对a[right + 1] ~ a[n -1]的影响，这样操作后得出的原数组a就是符合要求的原数组。

差分数组的初始化

​ 因为对差分数组的操作会出现right + 1，所以创建差分数组时要比原数组多分配一个空间，这样就不用特殊处理边界问题了。

​ 差分数组的创建方法有很多，首先最直观的创建方式就是根据前面提到的公式创建。

//根据差分数组的定义用公式求
void init(vector<int>& a)
{
    int n = a.size();
    vector<int> d(n + 1, 0); //比原数组多分配一个空间
    d[0] = a[0]; //0位置特殊处理一下
    for(int i = 1; i < n; i++)
        d[i] = a[i] - a[i - 1];//直接套公式求出差分数组
}

​ 第二中创建的方式更方便一些，就是将差分数组d视作一个都为0的数组，然后再依次将a[i]添加到[i, i]位置，代码如下：

//对差分数组操作
void add(vector<int>& d, int l, int r, int c)
{
    //直接用结论对差分数组操作即可
    d[l] += c;
    d[r + 1] -= c;
}
void init(vector<int>& a)
{
    int n = a.size();
    vector<int> d(n + 1, 0); //比原数组多分配一个空间
    for(int i = 0; i < n; i++)
        add(d, i, i, a[i]); //直接将a[i]加到[i,i]位置
}

​ 前面介绍的两种创建方式得到的结果是一样的，个人更喜欢使用第二种方式。

代码

​ 下面看一下整体的代码实现：

void add(vector<int>& d, int l, int r, int c)
{

    d[r + 1] -= c;
}


//第i个操作就把原数组a在[lefts[i], rights[i]]范围内的数都加一，返回对原数组多次操作后的最终数组
vector<int> solution(vector<int>& a, vector<int>& lefts, vector<int>& rights)
{
    int n = a.size(), q = lefts.size(); //获取数组长度和查询次数
    //构建差分数组
    vector<int> d(n + 1, 0);
    for(int i = 0; i < n; i++)
        add(d, i, i, a[i]);
    //依次对原数组操作
   	for(int i = 0; i < q; i++)
    	add(d, lefts[i], rights[i], 1);
    //对差分数组求前缀和得到修改后的数组
    int s = 0;
    vector<int> ans(n);
    for(int i = 0; i < n; i++)
    {
        s += d[i];
        ans[i] = s;
    }
    return s;
}

二维差分

​ 和一维差分类似，二维差分数组d（下标从0开始）的二维前缀和能得到对应修改后的二维数组a。现在假设要使用二维差分实现对左上角(r1,c1)和右下角(r2,c2)组成的矩形内都加上常数c，分析方法和一维一样：首先对d[r1] [c1]加c，会影响其右下角所有格子，即下图中蓝色区域；但是我们只想让红色区域加c，所以就可以把下边和右边的橙色区域都减c，以抵消影响；同时会发现右下角区域减了两次，所以最后要再把右下角多减的加回来。

​ 对应更新公式为：

  • d[r1] [c1] += c //蓝色区域都加c
  • d[r2 + 1] [c1] -= c //下侧橙色区域减c
  • d[r1] [c2 + 1] -= c //右侧橙色区域减c
  • d[r2 + 1] [c2 + 1] += c //右下角多减了一次，要加回来

​ 对应代码如下：

}

//每次修改都让(leftTops[0], leftTops[1])和(rightBottoms[0], rightBottoms[1])矩阵范围内加1
vector<vector<int>> solution(vector<vector<int>>& a, vector<vector<int>>& leftTops, vector<vector<int>>& rightBottoms)
{
    int m = a.size(), n = a[0].size(), q = leftTops.size();
    //构建初始二维差分数组
    vector d(m + 1, vector<int>(n + 1, 0));
    for(int i = 0; i < m; i++)
        for(int j = 0; j < n; j++)
            add(d, i, j, i, j, a[i][j]);
   	
    //在二维差分数组上进行修改操作
   	for(int i = 0; i < q; i++)
    {
        add(d, leftTops[0], leftTops[1], rightBottoms[0], rightBottoms[1], 1); //操作
    }
    
    //用二维前缀和的方法还原二维差分数组
    vector preSum(m + 1, vector<int>(n + 1, 0));
    for(int i = 0; i < m; i++)
        for(int j = 0; j < n; j++)
        {
            //preSum[i+1][j+1]就是修改后的a[i][j]
    		preSum[i + 1][j + 1] = preSum[i + 1][j] + preSum[i][j + 1] - preSum[i][j] + d[i][j];
        }
}

​ 前面的写法在计算二维差分数组的时候需要再开辟一个数组存储前缀和，但是如果我们在二维前缀和外围加上一圈0，就能够直接在差分数组上原地修改，不用额外使用一个数组了。这个和前缀和的存储方式类似，前缀和中preSum[i + 1] [j + 1]对应的是(i, j)位置；于是我们也让差分数组中d[i + 1] [j + 1]对应(i, j)位置，这样可以少处理很多边界条件，写起来比较方便。代码和前一种整体相同，只是在二维差分数组的插入位置有点改变。

void add(vector<vector<int>>& d, int r1, int c1, int r2, int c2, int c)
{
    d[r1] [c1] += c;
    d[r2 + 1] [c1] -= c;
    d[r1] [c2 + 1] -= c;
    d[r2 + 1] [c2 + 1] += c;
}

树状数组

​ 假设一种情况，如果有n次操作，每次操作都会有两种情况，一个操作是修改某一个数的值，第二个操作是查询某一范围内的和。如果使用前缀和来解决的话，每次修改过后就需要更新前缀和数组，这样最坏情况下时间复杂度是O(n^2)的。如果使用树状数组的话，可以以O(logn)的时间复杂度在修改某个值后更新维护的数组，以O(logn)的时间复杂度查询某一范围内的和，这样时间复杂度就变为了O(nlogn)。

Trie(前缀树/字典树)

​ Trie树数据结构可以快速的查询在已经出现过的字符串中某个字符串的数目，或者是以某个字符串为前缀的字符串的数目，所以也被叫做前缀树。

const int N = 1e5 + 9; //N的大小视数据范围而定，如果数据范围比较大，则需要进行一些额外的处理才可以，不然静态数组太大会栈溢出
int tr[N][26]{}, p[N]{}, e[N]{};
int cnt = 1;

//向树中添加字符串
void insert(string& str)
{
    int cur = 1;
    p[cur]++;
    for(char ch : str)
    {
        if(tr[cur][ch - 'a'] == 0)
            tr[cur][ch - 'a'] = ++cnt;
        cur = tr[cur][ch - 'a'];
        p[cur]++;
    }
    e[cur]++;
}

//查找某个字符串是否在树中
bool find(string word) 
{
    int cur = 1;
    for(char ch : word)
    {
        cur = tr[cur][ch - 'a'];
        if(p[cur] == 0)
            return false;
    }
    return e[cur] > 0;
}

//查找树中是否存在前缀为prefix的字符串
bool startsWith(string prefix) 
{
    int cur = 1;
    for(char ch : prefix)
    {
        cur = tr[cur][ch - 'a'];
        if(p[cur] == 0)
            return false;
    }
    return true;
}

//从树中删除字符串(删除已经插入到树中的字符串)
void remove(string& str)
{
    int cur = 1;
    for(char ch : str)
    {
        if(--p[cur] == 0)
        {
            tr[cur][ch - 'a'] = 0;
            return;
        }
        cur = tr[cur][ch - 'a'];
    }
    end[cur]--;
}

时间复杂度O(n^2)

​ 时间复杂度为O(n^2)的排序算法大都比较简单粗暴，比较好理解，代码也不复杂。

选择排序

​ 选择排序过程如下：

  • 在[i, n - 1]范围内找到最小值，并将最小值放到 i 位置，然后在[i + 1, n - 1]范围内继续

​ 选择排序每轮都会找到一个最小值放到已经有序的数组后面。

void Sort(vector<int>& arr)
{
    int n = arr.size();
    for(int i = 0; i < n; i++)
    {
        //在[i, n - 1]范围内找到最小值
        int idx = i;
        for(int j = i + 1; j < n; j++)
        {
			if(a[idx] > a[j])
                idx = j;
        }
        //将最小值移动到i位置，下一轮从[i + 1, n - 1]开始继续找最小值
        swap(a[i], a[idx]);
    }
}

冒泡排序

​ 冒泡排序的过程可以描述为一下行为：

  • 在[0, i]范围内，通过相邻位置两两比较大小并交换位置，将最大值移动到 i 位置，然后在[0, i - 1]范围内继续

​ 也就是说冒泡排序每轮都会确定一个最大值。

//每轮选择一个最大值移动到末尾
//第一轮，将[0, n - 1]中的最大值通过相邻位置交换移动到n - 1位置
//第二轮，将[0, n - 2]中的最大值通过相邻位置交换移动到n - 2位置
//...
//一直到第n - 1轮，将[0,1]中的最大值移动到1位置，此时就完成了排序
void Sort(vector<int>& arr)
{
    int n = arr.size();
    for(int i = 0; i < n; i++)
    {
        for(int j = 0; j < n - i - 1; j++)
        {
            if(a[j] > a[j + 1])
                swap(a[j], a[j + 1]);
        }
    }
}

插入排序

​ 冒泡排序的过程可以描述为一下行为：

  • [0, i]范围内已经有序，i + 1位置新来的数要在前面已经有序的数组中找到合适的位置插入，使得[0, i + 1]位置有序，然后继续处理后面的数

​ 不同于选择排序那种每次找到一个数直接放到对应位置，插入排序总是为新加入的数在已经有序的序列中找到合适的位置插入，使得整体依然有序。

void Sort(vector<int>& arr)
{
    int n = arr.size();
    //每次将a[i]加入到前面已经有序的序列中
    for(int i = 0; i < n; i++)
    {
        //在[0, i - 1]的有序序列中给a[i]找到合适的位置插入进入，使得新序列仍然有序
        for(int j = i - 1; j >= 0 && a[j] > a[j + 1]; j--)
        {
            swap(a[j], a[j + 1]);
        }
    }
}

时间复杂度O(nlogn)

​ 时间复杂度为O(nlogn)的排序算法就开始有一些比较厉害的操作了，相比于O(n^2)的排序算法，能减少很多没必要的比较。

归并排序

​ 归并排序采用“分治”的策略，下面简单描述一下：

  • 首先整个数组均分为左右两部分，使得左右两部分分别有序之后，再合并为一个有序的序列，就得到了排序好的数组
  • 对于左右两个部分，每部分又可以各自再划分为左右两个部分，将左右两部分排序后再合并得到有序的序列

​ 如下图所示，整个序列可以层层划分为一棵树状的结构，从叶子节点开始向上两两合并，使得父节点变为一个更大的有序的序列，然后再和其兄弟继续向上合并，直至合并到根节点。

​ 可以发现，由于每次都会把每部分分为两部分，所以整棵树的高度为logn，而且我们每一层都会进行合并，而每层的合并操作是O(n)的，所以归并排序整体的时间复杂度为O(nlogn)。并且最坏情况下仍然是O(nlogn)的，只是需要一个额外的辅助数组来帮助合并，所以空间复杂度为O(n)。

​ 看到这个描述过程，很容易会想到使用递归，所以下面就来介绍一下递归写法：

//归并排序递归实现
const int N = 1e5 + 9;
int a[N], h[N]; //a是原数组，h是辅助数组，用来实现合并操作
int n; //数组长度

//合并[l, m]和[m + 1, r]
void merge(int l, int m, int r)
{
    int i = l, j = m + 1, cnt = l;
    while(i <= m && j <= r)
    {
        if(a[i] <= a[j])
            h[cnt++] = a[i++];
        else
            h[cnt++] = a[j++]; 
    }
    
    while(i <= m)
        h[cnt++] = a[i++];
    while(j <= r)
       h[cnt++] = a[j++];
    
    //将合并好的数组放回原数组
    for(int k = l; k <= r; k++)
        a[k] = h[k];
}


void mergSort(int l, int r)
{
    //递归结束边界条件，如果只有一个数或者左边界大于右边界时不用合并，直接返回
    if(l >= r) return;
    
    int m = (l + r) >> 1;
    
    //让左右两边分别有序
    mergSort(l, m);
    mergSort(m + 1, r);
    
    //再合并两部分
    merge(l, m, r);
}

堆排序

//大根堆写法，小根堆写法类似，只要注意一下子节点和父节点的大小关系即可
const int N = 1e5 + 9;
int h[N], cnt = 0; //cnt表示堆内元素的个数，堆数组的有效下标为[1, cnt]，每次插入新元素时要先将cnt加一，所以是h[++cnt] = x

void up(int i)
{
    while(i / 2 && h[i] > h[i / 2])
    {
        swap(h[i], h[i / 2]);
        i /= 2;
    }
}

void down(int i)
{
    int l = 2 * i, r = 2 * i + 1;
    int best = i;
    if(l <= cnt && h[best] < h[l]) best = l;
    if(r <= cnt && h[best] < h[r]) best = r;
    if(best == i) return;
    swap(h[best], h[i]);
    down(best);
}

void push(int x)
{
    h[++cnt] = x; //下标从1开始，有效下标为[1, cnt]，每次插入新元素时要先将cnt加一，所以是h[++cnt] = x
    up(cnt);
}

int pop()
{
    int res = h[1];
    h[1] = h[cnt--]; //cnt就是堆数组中的最后一个位置，所以是h[cnt--]，先取出来再减一
    down(1);
    
    return res;
}

//cnt为0时说明堆是空的
bool empty()
{
    return cnt == 0;
}

全部组合

​ 如果数组中没有重复数字的话，那么写法就和上面的求所有子序列一样了，并且由于没有重复数字，连去重都不需要，只需要考虑每个位置要与不要即可。

vector<vector<int>> ans; //保存答案
vector<int>p; //记录搜索过程中的路径

//nums数组中无重复元素
void dfs(vector<int>& nums, int idx)
{
    if(idx == nums.size())
    {
        ans.push_back(p);
        return;
    }

    p.push_back(nums[idx]);
    dfs(nums, idx + 1);
    p.pop_back();

​ 但是如果有重复元素的话，如果我们再使用set对数组来去重的话，时间复杂度是很高的，而且对重复元素进行同样的展开会造成很多的重复操作。由于组合和数字的顺序无关，只和数字的数量有关。对于组合{1, 2, 2}和{2, 1, 2}，这两个就是一个组合，因为里面都有两个2和一个1，和顺序无关（和顺序有关的是排列）。

​ 因此我们可以先将数组进行排序，使得相同的数字都在一起，然后在讨论每种数字选择几个的策略，这样就可以很好的去重，避免了很多无用的递归展开。

vector<vector<int>> ans;
vector<int> p;

//nums中可能会有重复数字，nums传入前需要先排序
void dfs(vector<int>& nums, int i)
{
    if(i == nums.size())
    {
        ans.push_back(p);
        return;
    }
    //排序后假设数组为[1,1,1,2,2,3,4,4,5]，该循环就是要找到下一个数字的开始位置，比如当前数字是1，就要找到数字2的开始位置
    int j = i + 1;
    while(j < nums.size() && nums[j] == nums[i]){j++;}
    
    //在选0个当前数字的情况下，接着开始选下一个数字
    dfs(nums, j);
    //在选择x个当前数字的情况下，接着开始选下一个数字
    for(int k = i; k < j; k++)
    {
        for(int t = i; t <= k; t++) p.push_back(nums[i]); //路径添加对于数量的当前数字
        dfs(nums, j);
        for(int t = i; t <= k; t++) p.pop_back(); //回溯，返回原现场
    }
}

全排列

​ 排列和组合不同，不同的顺序也会是两个不同的排列，对于{1, 2, 2}和{2, 1, 2}来说，顺序不一样，所以是两个不同的排列。

​ 首先说一下数组中没有重复元素的情况，这种情况的求解比较简单，假设数组中有n个数，那么第一个位置有n中选择，然后第二个位置有n - 1中选择，依次递归下去。

//写法1：开一个额外的数组记录哪些位置的数字已经被选过了
vector<vector<int>> ans;
vector<int>p;
bool visit[10]{}; //visit[i]表面数字nums[i]当前是否已经被选过了

void dfs(vector<int>& nums)
{
    if(p.size() == nums.size())
    {
        ans.push_back(p);
        return;
    }
    for(int k = 0; k < nums.size(); k++)
    {
        if(!visit[k])
        {
            visit[k] = true;
            p.push_back(nums[k]);
            dfs(nums);
            p.pop_back();
            visit[k] = false;
        }
    }
}



//写法2：不用开额外数组，循环中让[i, n - 1]位置的数字和i位置的数字交换，以此实现i位置选择不同的数。不需要辅助数组，代码更简洁，但是不好理解
void dfs(vector<int>& nums, int i)
{
    if(i == nums.size())
    {
        //来到结束位置的时候，nums数组已经通过交换得到了一个排列了，直接保存答案
        ans.push_back(vector<int>(nums.begin(), nums.end()));
        return;
    }
    //当前的i位置，从[i, n - 1]的每个数都能在位置i，遍历所有可能
    for(int k = i; k < nums.size(); k++)
    {
        swap(nums[i], nums[k]); //i位置和k位置数字交换
        dfs(nums, i + 1);
        swap(nums[i], nums[k]); //回退到原始状态，如果前面原数组1位置和3位置交换了，此时必须回退到原始状态，方便下一次循环时让1位置和4位置继续交换
    }
}

​ 下面再说一下数组中可能会有重复数字的情况，去重的写法也比较简单，这里基于交换的写法进行修改。只需要交换的时候，保证交换到 i 位置的数字是不重复的即可，所以在交换的循环中记录已经交换到 i 位置的数字，如果该数字之前就已经被交换到 i 位置了，就不再交换了，只交换没有换到过 i 位置的数。

vector<vector<int>> ans;

void dfs(vector<int>& nums, int i)
{

    {
        //来到结束位置的时候，nums数组已经通过交换得到了一个排列了，直接保存答案
        ans.push_back(vector<int>(nums.begin(), nums.end()));
        return;
    }
    //当前的i位置，从[i, n - 1]的每个数都能在位置i，遍历所有可能

    for(int k = i; k < nums.size(); k++)

        if(!record.count(nums[k]))

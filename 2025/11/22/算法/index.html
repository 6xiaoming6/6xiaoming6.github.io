整数二分

二分法用于在有序序列中快速查找，由于每次都会跳过一半的数据，所以时间复杂度为O(logn)。下面写一个通用的整数二分模板，该模板会返回序列中第一个>=目标数target的数据的位置。

所以这是一个万能模板，会返回序列中第一个>=目标target的数的索引值，查找不到就返回数组长度。不论是找唯一数据的索引值、重复数据的左边界、大于target的第一个数的索引值（target不是数组的的某个值，但在数组的最大最小值之间），都可以直接用这个模板。

需求 写法 如果不存在
≥ x 的第一个元素的下标 lowerBound(nums, x) 结果为 n
> x 的第一个元素的下标 lowerBound(nums, x + 1) 结果为 n
< x 的最后一个元素的下标 lowerBound(nums, x) − 1 结果为 −1
≤ x 的最后一个元素的下标 lowerBound(nums, x + 1) − 1 结果为 −1
需求(数组下标从0开始) 写法
< x 的元素个数 lowerBound(nums, x)
≤ x 的元素个数 lowerBound(nums, x+1)
≥ x 的元素个数 n−lowerBound(nums, x)
> x 的元素个数 n−lowerBound(nums, x+1) 
//闭区间写法[l, r]
int lowerBound(vector<int>nums, int target)
{
    //这里我们要返回的索引值范围是[left, right]，都是闭区间，所以当left > right时该区间为空，说明查找结束
    int left = 0, right = nums.size() - 1;
    while(left <= right)
    {
        //防止left + right数据溢出
        int mid = left + (right - left) / 2;
        if(nums[mid] >= target)right = mid - 1;
        else left = mid + 1;
    }
    return left;
}



//开区间写法(l, r)
int lower_bound(vector<int>& nums, int target) 
{
    int left = -1, right = nums.size(); // 开区间 (left, right)
    while (left + 1 < right) { // 区间不为空
        //防止下标溢出
        int mid = left + (right - left) / 2;
        if (nums[mid] >= target) {
            right = mid; // 范围缩小到 (left, mid)
        } else {
            left = mid; // 范围缩小到 (mid, right)
        }
    }
    // 循环结束后 left+1 = right
    // 此时 nums[left] < target 而 nums[right] >= target
    // 所以 right 就是第一个 >= target 的元素下标
    return right;
}

    {
        //防止left + right数据溢出
        int mid = left + (right - left) / 2;
        if(nums[mid] >= target)right = mid - 1;
        else left = mid + 1;
    }
    return left;
}
//假设查找的数据都是合理的，找不到会返回len
//如果序列中数据唯一，不重复，那么最终返回target的索引值
nums = [1, 2, 3, 4, 5, 6, 7];
lowerBound（nums, 3) --->最终返回3的索引值2;
    
//如果序列中数据可以重复，那么最终返回target的左边界的索引值（第一个 >= target的数的位置）
nums = [1, 2, 2, 3, 3, 3, 4, 5, 6, 7];
lowerBound（nums, 3) --->最终返回3的左边界3;

//如果要查找的数据在序列最大值和最小值范围内target∈[min, max]，会返回第一个 > target的数的位置
nums = [1, 6, 8, 10, 10, 12, 18, 25, 33];
lowerBound（nums, 11) --->最终返回第一个大于11的数12的位置5;

上面说到的都是找到>=target的位置，一般用于找target唯一索引 或者 target的左边界 或者 第一个>target（target不在数组中，但是在数组的最大最小值范围中）。

可以进行一些小转换，用于找到第一个>target的值的索引，和前面的不同的是原始的模板只有当target不在数组中时才会返回第一个>target的值的索引，而当target在数组中时无法处理，比如找到target的右边界，原始的模板无法处理。

//----------------------------------1.要找到 >target 的第一个数的索引
//如果target+1存在，那么自然会找到第一个target+1的位置，那么就是第一个>target的位置
//如果target+1不存在，那么后面找到的位置仍然是第一个>target的位置
lowerBound（nums, target + 1);

nums = [1, 2, 3, 4, 5, 9, 10];
lowerBound（nums, 4 + 1);//target=4，lowerBound(nums, 4+1)会找到第一个>=5的数5，其位置为4
lowerBound（nums, 5 + 1);//target=5，lowerBound(nums, 5+1)会找到第一个>=6的数9，其位置为5


//----------------------------------2.要找到 <target 的第一个数的索引
//如果target存在，那么自然会找到target的左边界，那么左边界-1就是第一个 <target 的第一个数的索引
//如果target不存在，那么后面找到的位置仍然是第一个>target的位置，此位置-1就是第一个 <target 的第一个数的索引
lowerBound（nums, target) - 1;

nums = [1, 2, 5，5, 9, 11];
//target=5，lowerBound(nums, 5)会找到的位置第一个>=5的数5，其位置为2，减一为1，而1就是符合要求的索引
lowerBound（nums, 5) - 1;
//target=4，lowerBound(nums, 4)会找到的位置第一个>=4的数5，其位置为2，减一为1，而1就是符合要求的索引
lowerBound（nums, 4) - 1;


//----------------------------------3.要找到 <=target 的最后一个数的索引
//如果target存在，会找到target的右边界
//如果target不存在，会找到最后一个<target的值的索引
lowerBound（nums, target + 1) - 1;

nums = [1, 2, 5，5, 9, 11];
//target=5，lowerBound(nums, 6)会找到nums[4]=9的位置，该位置减一为3，就是target的右边界
lowerBound（nums, 6) - 1;
//target=3，lowerBound(nums, 4)会找到nums[2]=5的位置，该位置减一为1，就是最后一个小于target的数的索引
lowerBound（nums, 4) - 1;

a * qpow(b, MOD - 2, MOD) % MOD

质数

质数是指一个大于1的自然数，除了1和自身外，不能被自然数整除的数叫做质数。比如2，3，5，7，11，13等都是质数。

判断质数

下面是一个比较简单的判断一个数是不是质数的算法，当n比较小的时候可以使用，因为其时间复杂度为O($\sqrt{n}$)。

bool isPrime(int n)
{
    for(int i = 2; i * i <= n; i++)
        	if(x % i == 0)
                return false;
    return true;
}

试除法分解质因数

当给定一个数字n时，求出n的所有质数因子。

vector<int> solve(int n)
{
    vector<int> ans;
    for(int i = 2; i * i <= n; i++)
    {
        if(n % i == 0)
        {
            while(n % i == 0)
            {
                n /= i;
                ans.push_back(i);
            }
        }
    }
    if(n > 1)ans.push_back(n);
    
    return ans;
}

质数筛

给定一个数字n，返回1~n范围内的所有质数的数量。

这个问题如果遍历从2到n，再分别判断每个数是不是质数，那时间复杂度是O(n$\sqrt{n}$)，是很容易超时的，而质数筛就比较快了。

共有埃氏筛和线性筛(欧拉筛)两种方法，前者时间复杂度为O(nlog(logn))，后者顾名思义时间复杂度为O(n)。一般情况下埃氏筛够用了，但是欧拉晒能表现的更好。

//埃氏筛的做法可以简单理解为把每一个质数的倍数都排除掉，最后留下来的就是质数
int getPrimeCount(int n)
{
    int prime[N];//记录所有的质数
	bool visit[N];//false表示为质数，true表示为合数，初始时里面的值均为false
    
    int cnt=0;
    for(int i = 2; i <= n; i++)
    {
        //第一次进入循环的i为2
        if(!visit[i])
        {
            prime[cnt++] = i;//找到了一个质数，存到数组里
            //遍历4，6，8，10，12...，将2的整数倍都排除
            for(int j = i * i; j <= n; j += i)
                visit[j] = true;
        }
    }
    return cnt;
}



//在埃氏筛的循环中，当i = 2时，会排除4，6，8，10，12...；当i = 3时，会排除9，12，15....。可以看到二者都重复排除了12这个数，所以埃式筛会重复的排除数字
//而欧拉晒则对这一问题进行了优化，只会保证每个数只会被过一遍，没有重复的情况，所以时间复杂度是线性的。二者代码差异不是很大，只是最内层的循环发生了改变
int getPrimeCount(int n)
{
    int prime[N];//记录所有的质数
	bool visit[N];//false表示为质数，true表示为合数，初始时里面的值均为false
    
    int cnt=0;
    for(int i = 2; i <= n; i++)
    {
        //第一次进入循环的i为2
        if(!visit[i])
        {
            prime[cnt++] = i;
            for(int j = 0; primes[j] <= n / i; j++)
            {
                visit[primes[j]*i] = true;
                if(i % primes[j] == 0) break;
            }
        }
    }
    return cnt;
}

滑动窗口

滑动窗口使用前提：

  1. 连续子数组/子串。（注意不是子序列，必须是连续的子串或者子数组）
  2. 有单调性，即所求的目标会随着窗口的变化单调变化。例如正数数组，或者负数数组，窗口内数组的和会随着数组的大小变化单调变化；但是如果数组内有正有负的话，就不满足单调性了，就无法使用滑动窗口解决。

定长窗口

定长窗口的题目求解目标一般明确指出要求长度为k的符合要求的连续子数组或者子串。可以直接从0到n - 1枚举所有右端点，枚举所有以i位置为右端点的窗口，从中找到符合要求的答案。

下面是算法模板：

int solution(vector<int>& nums, int k)
{
    int n = nums.size(), left = 0, ans = 0;
    //枚举所有右端点
    for(int i = 0; i < n; i++)
    {
        //右端点进入窗口
        ...;//计算
        
        //因为要求长度为k，所有窗口大小为k时才符合要求
        if(i - left + 1 < k)
            continue;
        
        //此时窗口大小已经为k了，就找到了一个长度为k的窗口
        ...;//计算该窗口，更新结果ans
        
        left++;//左端点向前移动
    }
    return ans;
}

动长窗口

动长窗口的题目一般不会对子数组或者子串的长度进行限制，一般会要求出满足某种限制条件的子数组或子序列，这种情况下窗口的大小就是不固定的。

下面是算法模板：

int solution(vector<int>& nums, int k)
{
    int n = nums.size(), left = 0, ans = 0;
    //枚举所有右端点
    for(int i = 0; i < n; i++)
    {
        //右端点进入窗口
        ...;//计算
        
        //当窗口内的子数组或者子序列不满足条件时，循环让左端点移出窗口，直到满足限制条件
        while(!限制条件)
        {
            //左端点移出窗口，直到该窗口内地子数组或者子序列满足限制条件
            ...;//执行左端点移出的更新操作
            left++;
        }
        
        //此时的窗口是满足题目中限制条件的
        ...;//计算该窗口，更新结果ans
    }
    return ans;
}

高精度

高精度加

问题：两个长度为1000（或者更长，以至于无法用某一个类型表示它）的大正整数A和B相加，输出A+B的结果。

显然这么多位的数字，我们无法用任何一个数值类型变量存储，所以就需要使用高精度加方法来模拟计算。高精度加就是使用一个数组存储大整数的每一位，然后模拟加法运算。

#include <vector>
#include <string>
#include <iostream>
#include <algorithm>
using namespace std;

int main()
{
    string a, b;
    cin>>a>>b;
    
    //输入，因为要从个位开始加，所以逆序存储大整数
    vector<int>va, vb;
    for(int i = a.size() - 1; i >= 0; i--) va.push_back(a[i] - '0');
    for(int i = b.size() - 1; i >= 0; i--) vb.push_back(b[i] - '0');
    
    //计算，向前进位
    vector<int> ans;
    int t = 0; //t承担进位，最低位没有进位，初始值为0
    for(int i = 0 ; i < va.size() || i < vb.size(); i++)
    {
        if(i < va.size()) t += va[i];
        if(i < vb.size()) t += vb[i];
        ans.push_back(t % 10);
        t /= 10; //如果当前位相加的值 >= 10则t/10 == 1，表示向前进位，否则t/10 == 0，不会向前进位
    }
    //判断最高位是否还需要向前进位
    if(t) ans.push_back(t);
    
    //逆序输出
    for(int i = ans.size() - 1; i >= 0; i--)
        cout << ans[i];
    cout << endl;
    
    return 0;
}

矩阵对角线

​ 使用该模板可以快速遍历矩阵的对角线，假设我们的矩阵大小为m x n，即m行n列。

​ 首先来看第一种情况，对角线是左上到右下方向的。对于每一条对角线上的元素mat[ i ] [ j ]，i - j都是相同的值，于是我们可以发现这样一个式子：

k = i - j + n，这个 k 表示从右上到左下方向上第 k 条对角线（下图中红色数字就是k的值），k的取值是[1, m + n - 1]，所以我们只需要遍历k的所有取值就能遍历所有的对角线。

​ 对于第k条对角线来说，我们只需要确定元素的列坐标 j ，也就能根据等式 k = i - j + n 求出行坐标 i ，根据等式可以判断，i最小时j最小，i最大时j最大：

  • i 最小值为0时， minj = n - k。但是要保证minj >= 0，所以 minj = max(n - k, 0)
  • i 最大值为m - 1时， maxj = m - 1 - k + n。但要保证 maxj <= n - 1，所以maxj = min(m - 1 - k + n, n - 1)

​ 这样以来我们就可以在遍历每条对角线的时候求出 j 的取值范围，进而根据 k = i - j + n 求出 i ，从而找到该对角线上的所有元素mat[ i ] [ j ]。下面看看代码怎么写：

void matDiagonal(vector<vector<int>>& mat)
{
    //确定行数和列数
    int m = mat.size(), n = mat[0].size();
    
    //遍历所有对角线，k的取值范围为[1, m + n - 1]
    for(int k = 1; k < m + n; k++)
    {
        //根据 j = i - k + n 确定该对角线上 j 的取值范围
        int minj = max(n - k, 0); //i = 0时j最小，但时不能小于0
        int maxj = min(m - 1 - k + n, n - 1); //i = m - 1时j最大，但是不能大于n - 1
        
        //遍历对角线上的所有元素
        for(int j = minj; j <= maxj; j++)
        {
            int x = mat[j + k - n][j]; //根据公式 i = j + k - n 求出对应的行坐标 i
        }
    }
}

​ 下面介绍第二种情况，对角线是右上到左下方向的。对于每一条对角线上的元素mat[ i ] [ j ]，i + j都是相同的值，于是我们可以发现这样一个式子：

k = i + j，这个 k 表示从左上到右下方向上第 k 条对角线（下图中红色数字就是k的值），和上面一种情况不同的是，此时k的取值是[0, m + n - 2]，所以我们只需要遍历k的所有取值就能遍历所有的对角线。

​ 对于第k条对角线来说，我们只需要确定元素的列坐标 j ，也就能根据等式 k = i + j 求出行坐标 i ，根据等式可以判断，i最小时j最大，i最大时j最小：

  • i 最大值为m - 1时， minj = k - m + 1。但要保证 minj >= 0，所以minj = max(k - m + 1, 0)

  • i 最小值为0时， maxj = k。但是要保证minj <= n - 1，所以 maxj = min(k, n - 1)

​ 这样以来我们就可以在遍历每条对角线的时候求出 j 的取值范围，进而根据 k = i + j 求出 i ，从而找到该对角线上的所有元素mat[ i ] [ j ]。下面看看代码怎么写：

void matDiagonal(vector<vector<int>>& mat)
{
    //确定行数和列数
    int m = mat.size(), n = mat[0].size();
    
    //遍历所有对角线，k的取值范围为[0, m + n - 2]
    for(int k = 0; k < m + n - 1; k++)
    {
        //根据 j = k - i 确定该对角线上 j 的取值范围
        int minj = max(k - m + 1, 0); //i = m - 1时j最小，但时不能小于0
        int maxj = min(k, n - 1); //i = 0时j最大，但是不能大于n - 1
        
        //遍历对角线上的所有元素
        for(int j = minj; j <= maxj; j++)
        {
            int x = mat[k - j][j]; //根据公式 i = k - j 求出对应的行坐标 i
        }
    }
}
